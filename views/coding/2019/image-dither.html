                          # 图像抖动(dithering)的概念

                          图像抖动主要目的是使用有限的颜色提升图片的显示质量。其算法核心是误差扩散(error diffusion)。

                          假如现在有一台只支持黑/白两色的打印机，需要打印一张有灰阶的图片。

                          一般情况下，如果这个图片的色点接近 0 则打印黑色，如果接近 255 就打印白色，比如 96 比较接近 0，则认为这个点为黑色。

                          那么问题来了，如果碰到一连串都是 96 的灰点，那么打印出来会是一大块黑色，而这会使图像显示效果大打折扣，因为无法体现灰度。

                          误差扩散就主要改进这个问题，96->0，认为误差值是 96，在决定下一个点的颜色时会考虑到这个误差值。假如下一个点仍然是 96，那么加上误差值 96，则认为这个点为 192，192 比较接近 255，所以打印白色，同理，这个点的误差值为 192-255->-63,以此类推，下个点需要加上这个误差值。

                          我们可以看到经过误差值计算后的图片显示，由于我们只是将误差值往右边推算，而向下并没有推算，所以效果还是不太行。

                          # 二维扩散(Two-Dimensional Error Diffusion Dithering)

                          一般图像抖动算法都是向右，向下计算误差，而且从不往后计算，因为如果往后计算的话，就会形成死循环。

                          # Floyd-Steinberg Dithering

                          Floyd-Steinberg Dithering 是最为著名的误差扩散公式，由 Robert Floyd 和 Louis Steinberg 在 1976 年提出。它的公式如下：

                                  X       7/16
3/16    5/16    1/16
                          1
                          2

                          其中的 X 代表当前的点，公式把误差分散到 X 的周边 4 个点，假如 X=96，则误差为 96,那么四个点的值需要分别加上 误差乘以误差率，得出以下的值：

                                  X       +42
+18    +30      +6
                          1
                          2

                          • 边界问题：比如图片最右边的点是没有右邻的，比如图片的最后一行是没有下邻的，需要判断边界。
                          • findClosestPaletteColor:这个取决于你转换后的图片需要几种颜色。我这边采用黑白红三色。
                          const palette = [
  [0, 0, 0],
  [255, 255, 255],
  [255, 0, 0],
];

function findClosestPaletteColor(color) {
  let closest = palette[0];
  for (let i = 0; i < palette.length; i++) {
    if (diff(color, palette[i]) < diff(color, closest)) {
      closest = palette[i];
    }
  }

  return closest;
}

function diff(color, paletteColor) {
  let Rdiff = color[0] - paletteColor[0];
  let Gdiff = color[1] - paletteColor[1];
  let Bdiff = color[2] - paletteColor[2];
  let distanceSquared = Rdiff * Rdiff + Gdiff * Gdiff + Bdiff * Bdiff;
  return distanceSquared;
}

                          • node 的实现
                          //Floyd-Steinberg Dithering
const palette = require("./palette.js");

module.exports = function(pixels) {
  let w = pixels.shape[0];
  let h = pixels.shape[1];

  for (let y = 0; y < h; y++) {
    for (let x = 0; x < w; x++) {
      oldRGB = palette.getRGB(x, y, pixels);

      newRGB = palette.findClosestPaletteColor(oldRGB);

      errRGB = palette.calcError(oldRGB, newRGB);

      palette.setRGB(x, y, pixels, newRGB);

      if (x + 1 < w) {
        calcRGB = palette.getRGB(x + 1, y, pixels);
        effectRGB = palette.diffusion(calcRGB, errRGB, 7.0 / 16);
        palette.setRGB(x + 1, y, pixels, effectRGB);
      }

      if (x - 1 >= 0 && y + 1 < h) {
        calcRGB = palette.getRGB(x - 1, y + 1, pixels);
        effectRGB = palette.diffusion(calcRGB, errRGB, 3.0 / 16);
        palette.setRGB(x - 1, y + 1, pixels, effectRGB);
      }

      if (y + 1 < h) {
        calcRGB = palette.getRGB(x, y + 1, pixels);
        effectRGB = palette.diffusion(calcRGB, errRGB, 5.0 / 16);
        palette.setRGB(x, y + 1, pixels, effectRGB);
      }

      if (x + 1 < w && y + 1 < h) {
        calcRGB = palette.getRGB(x + 1, y + 1, pixels);
        effectRGB = palette.diffusion(calcRGB, errRGB, 1.0 / 16);
        palette.setRGB(x + 1, y + 1, pixels, effectRGB);
      }
    }
  }

  return pixels;
};

                          # Further

                          • 影响抖动算法的因素：
                          1. 计算误差值时是否采用整数还是小数计算，整数可能会丢失一些精度。
                          2. 用 127 还是 128 来判断黑白，在不同的图片上效果不同。
                          3. 彩色图片，luminance（亮度）的计算方式会影响很大，HSL luminance formula ( [max(R,G,B) + min(R,G,B)] / 2). Others use ([r+g+b] / 3) or one of the ITU formulas. YUV or CIELAB will offer even better results.（不太了解）
                          4. Gamma 纠正和图片的预处理
                          5. 计算路径的方向：一般采用 left->right, top->bottom,但有的算法是，第一行 left->right,第二行 right->left,不断交替，这可能效果更好，但复杂一点。
