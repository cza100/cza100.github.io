# 前言

最近项目某个地方准备用 promise 或 async/await 重写，正好再整理一下 Promise 和 Async/Await 的相关知识,基本概念不再赘述。

# 1. Promise

# 1.1 三种状态

  • pending: 初始状态，直至 relove 或 reject
  • resolve: 成功返回
  • reject: 失败返回

# 1.2 链式调用

Promise 链式调用的本质是往下传递上一个 Promise，并且用 then 接收。

promiseFunc_1()
  .then((result) => {
    console.log(result);
    return promiseFunc_2();
  })
  .then(
    (result) => {
      console.log(result); // promiseFunc_2的resolve值
    },
    (err) => {
      console.log(err); // promiseFunc_2的reject值
    }
  );

可以看到，then 的两个参数分别是：

  • 处理上一步 Promise 的 resolve 状态的回调函数
  • 处理上一步 Promise 的 reject 状态的回调函数

# 1.3 catch 与 then 的顺序

reject 的值，一定会进入 then 中的第二个回调，如果 then 中没有写第二个回调，则进入 catch。

resolve 的值，一定会进入 then 的第一个回调，肯定不会进入 catch。

若 catch 比 then 写在链式写法的前面，则 catch 优先捕获错误。

promiseFunc_1()
  .then((result) => {
    console.log(result);
    return promiseFunc_2();
  })
  .catch((err) => {
    console.log("优先捕获错误");
  })
  .then(
    (result) => {
      console.log(result);
    },
    (err) => {
      console.log(err);
    }
  );

# 1.5 跳出或终止链式 Promise

then,catch 都不能 break,链式后面会继续执行

promiseFunc_1()
  .then((result) => {
    console.log(result);
    return promiseFunc_2();
  })
  .catch((err) => {
    console.log("优先捕获错误");
  })
  .then((result) => {
    console.log(result); // 这一步会继续执行
  });

# 1.6 Promise 数组

一个 Promise 数组或者 Iterator，可以同时处理多个 Promise。

  • Promise.all

并行执行，如全部 resolve 则返回 resolve 值的数组，如碰到 reject,则返回 reject 的值，并且立即退出，不等待其他 Promise 的结束。

  • Promise.race

返回第一个 resovel/reject 的值。

  • Promise.allSettled

只有当所有 Promise 都返回状态，才会执行下一步操作。

# 3. 串行获取数据

我们知道，Promise 数组用 Promise.all()等运行都是并行的，怎么做到串行输出呢？

// 同理
const promiseArr = [];

promiseArr.push(promiseFunc_2()); // promiseFunc_2已开始

const promiseArr = [promiseFunc_1, promiseFunc_2, promiseFunc3];

for (let i = 0; i < promiseArr.length, i++) {

此代码逻辑为将条形码、二维码的 fabric object 生成为图片添加到 canvas 中，执行以上代码，发现条形码、二维码并没有按预期生成。

原因是 qrCode.toDataURL()、fabric.Image.fromURL()都是异步操作，所有当 canvas 重新渲染的时候，两个回调函数并没有执行完毕。

接下来使用异步改写，这里考虑到 Await 只能接受 Promise，上面代码结构明显适合使用 Promise。以下为改写的代码。

canvas.loadFromDatalessJSON(json, handle.bind(null, args));

const handle = (args) => {
  const promiseArr = [];
  let objects = canvas.getObjects();
  objects.foreach((item, index) => {
    // 处理barcode
    if (item["type"] == barcode) {
      jsBarcode(c, item["text"]);
      const p = new Promise((resolve) => {
        fabric.Image.fromURL(c.toDataURL(), (img) => {
          canvas.add(img);
        });
      });
      promiseArr.push(p);
    }
    // 处理qrcode
    if (item["type"] == qrcode) {
      const p = new Promise((resolve) => {
        qrCode.toDataURL(
          target,
          {
            errorCorrectionLevel: "Q",
            scale: 1,
            version: version,
            margin: 0,
            color: { dark: dark, light: light },
          },
          function(err, url) {
            fabric.Image.fromURL(url, function(img) {
              canvas.add(img);
              resolve();
            });
          }
        );
      });
      promiseArr.push(p);
    }
  });

  // canvas重新渲染并生成图片
  Promise.all(promiseArr).then(() => {
    canvas.renderAll();
    let stream = canvas.createPNGStream();
    stream.on("data", (chunk) => {
      res.write(chunk);
    });
    stream.on("end", () => {
      res.end();
    });
  });
};
